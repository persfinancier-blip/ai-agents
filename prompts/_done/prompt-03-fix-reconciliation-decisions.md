## Контекст решений

Первый кодовый срез — **Goal Entity (дерево декомпозиции) + ресурсы на узлах + детерминированная увязка** (арифметика, без LLM). Аналитические справки офисных агентов через LLM — отдельный поздний срез (roadmap M7–M8). Под этот scope §7 решён так:

| # | Вопрос | Решение |
|---|---|---|
| Q1 | Формулы увязки | **Детерминированное ядро сейчас.** Первый срез считает только арифметику: сумма детей vs родитель; сумма ресурсов узла vs доступно; разрыв = разница. LLM-оценка «реального потолка» и справки — отложены до среза офисных агентов. |
| Q2 | Единицы ресурсов | `resource.kind ∈ {people, money, other}`. **people** — загрузка юнита в % ; **money** — сумма в валюте; **other** — текстовое требование без числа (внимание/компетенции/инфраструктура). Расширяется позже. |
| Q4 | Порог определённости | **Авто-детекция** тумана/определённости системой: узел «в тумане», если нет KPI ИЛИ владельца ИЛИ данных; «определён» при наличии всех трёх. **Бэклог — только ручной флаг** (осознанное «не сейчас»), ставит человек. |
| Q7 | Триггер пере-увязки | **Пересчёт на каждое изменение** узла/ресурса (дерево небольшое, считать дёшево, всегда актуально). Расписание/ручной триггер — когда появятся дорогие LLM-справки. |
| Q3 | Права ролей на чужие узлы | **Отложено.** В первом срезе ярлыки собственник/менеджмент/исполнитель — **чистая семантика ответственности, без модели доступа.** Права появятся с аутентификацией/ролями (MVP-роли). |
| Q5 | Версионирование справок | **Отложено** до среза офисных агентов (справок пока нет). |
| Q6 | Каталог навыков | **Отложено** до среза навыков-воркфлоу (листовой слой). |
