# prompt-25 — chore: гигиена конфига и канона (пуш-ответственность, protect-main, graphify, handoff)

**Кому:** Claude Code
**Ветка:** новая `chore/config-hygiene` **от `main`**
**Тип коммита:** `chore:` (кросс-каттинг гигиена конфига/канона — один отдельный PR, по `commits.md`)
**Каноны:** `CLAUDE.md` («Процесс», «Экономия токенов»), `COWORK.md`, `.claude/rules/commits.md`. Продуктовый канон (PRD/Management_Model/ADR) НЕ затрагивается.

---

## Контекст (проверь сам)

Сессия-настройка выявила пять правок гигиены. Все — **записи в конфиг/канон**, продуктового кода не касаются. Cowork (архитектурный со-пилот) в репо не пишет — этот проход исполняет и пушит Claude Code.

## Scope

**Делаем (один PR):**

1. **Закрыть Bash-дыру `protect-main`.** Сейчас `.claude/hooks/protect-main.sh` навешен только на матчер `Edit|Write|NotebookEdit` (см. `.claude/settings.json` → `hooks.PreToolUse`), поэтому запись через Bash-`git` (commit/merge/push) в `main` проходит мимо охраны — так ранее и просочился прямой коммит в `main`. Навесить защиту и на `Bash`: если текущая ветка `main` И команда — git-запись (commit/merge/push в `main`) → блок `exit 2` с причиной в stderr. **Исключение:** активный merge (`.git/MERGE_HEAD` существует) — пропускать, чтобы не ломать легитимное разрешение merge-конфликтов. См. пункт про `protect-main` в `BACKLOG.md`.

2. **Ослабить graphify-принуждение.** В `.claude/settings.json` два `PreToolUse`-хука (матчер `Bash` для grep/find и матчер `Read|Glob`) подставляют текст «MANDATORY: you MUST run graphify before grepping/reading». Это **противоречит** `CLAUDE.md` («graphify — только для вопросов о связях; для точечных задач обычные grep/Read: дешевле») и жжёт токены. Убрать оба этих принуждающих хука (или свести к необязательной однострочной подсказке без «MANDATORY/MUST»). Сам скилл graphify, `graphify update` и прочие хуки — НЕ трогать.

3. **Гейт-фикс `handoff.md`.** В `.claude/commands/handoff.md`, блок «Гейт проверки», первый буллет («Поля «Состояние/Решения» бери ТОЛЬКО из фактического `git`/`DEVLOG`/`BACKLOG`…») разбить на два:
   - «Состояние» (ветка/коммит/готово-uncommitted) — ТОЛЬКО из фактического `git`/`DEVLOG`/`BACKLOG`; при ошибке git — писать `состояние не подтверждено: <ошибка>`, не выдумывать.
   - «Решения» — брать из текущего разговора (это решения, которых ещё НЕТ в файлах), но только реально принятые в сессии, не домысленные.

   Остальные буллеты гейта не трогать. Причина: Codex-ревью на PR #2 верно заметил, что старая формулировка запрещала брать «Решения» из разговора — а это ровно то, ради чего `/handoff` существует.

4. **Блок «Пуш и ответственность»** в `.claude/rules/commits.md` (кратко продублировать в `COWORK.md`):
   - Чтение/проверка репо — Cowork через GitHub MCP (**read-only**).
   - Записи (commit/branch/PR/merge) — **только Claude Code** (у Cowork нет права записи в репо).
   - Прямой пуш в `main` — только человек (владелец); агенты — всегда через PR.
   - Каждая строка-запуск от Cowork несёт префлайт (Шаг 0) и «хвост-уборку» (ниже).

5. **Формат промпта в `COWORK.md`.** Заменить старое правило «Cowork сохраняет промпт сразу в `prompts/`» на: Cowork **кладёт** промпт-файл в `prompts/`, но в репо **НЕ коммитит и НЕ пушит** — публикует Claude Code в рамках прохода. Добавить текст «Хвост-уборка»:

   > **Хвост-уборка (обязателен в каждом промпте для Claude Code):** промпт-файл этой задачи лежит в `prompts/prompt-NN-*.md` (положил Cowork, в репо не коммитит). Claude Code: (1) **Шаг 0 — префлайт:** `git status` + `git log origin/main..HEAD`; если дерево грязное или есть незапушенное — показать и развести осознанно (правильная ветка, Conventional-коммит, без `git add .`), открытые PR не мержить вслепую; (2) коммитит промпт-файл вместе с работой (или отдельным `docs:`/`chore:`, если срез кодовый); (3) после мержа переносит его в `prompts/_done/`; (4) завершает с чистым деревом (`git status` пуст, `origin` синхронизирован). Если что-то не запушилось — сообщить точную ошибку, задачу завершённой не считать.

**НЕ делаем:**
- `backend/`, `frontend/`, миграции, тесты — не трогаем.
- GitHub branch protection на `main` — **не включаем** (отдельное решение владельца, не в этом проходе).
- Логику/содержимое скилла graphify — не трогаем, только принуждающие хуки.
- Содержимого продуктового канона (PRD, Management_Model, ADR, Visual_Reference) не касаемся.
