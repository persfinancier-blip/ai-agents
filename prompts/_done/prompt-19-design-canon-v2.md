### 2. `Visual_Reference.md` — новая «Часть III. Дизайн-направление v2 (утверждено 2026-07-13)»
Зафиксировать (значения брать из эталонного HTML, не выдумывать):
- **Философия:** «полотно вместо рамок» — никаких панелей-коробок и меню-баров; зоны разделяются светом (гашение сетки, ореол), глубиной (ярусы теней), формой (прямоугольное = данные, круглое/рой точек = живое/разум) и территорией (слои не заезжают друг на друга). Плавающие пилюли вместо баров. Рабочие области пользователь собирает под себя (связка с `Workspace_Platform.md`).
- **Палитра:** фон `#0b0d13`; сетка `rgba(255,255,255,.032)` 34px; поверхность узла `rgba(24,26,34,.92)` + бордер `rgba(255,255,255,.09)`; чернила `#e7e8ee` и ступени прозрачности; идентичность агентов/веток — фиолет `#a855f7`, циан `#22d3ee`, оранж `#fb923c`, синий `#3b82f6`. **Светофор статусов (зелёный/жёлтый/красный) сохраняется** для health/риска/выполнено — идентичность ≠ статус, не смешивать.
- **Геометрия:** узлы/карточки — радиус 14px; иконки-бейджи — 10px; пилюли/чипы/инпуты — 99px; реплики — 16–18px с «хвостиком» 4–5px. Пунктир остаётся кодом «черновик/туман/нет данных».
- **Компоненты:** узел-карточка (иконка-бейдж с цветом ветки + имя + владелец + чип «N KPI»); орб-советник (canvas, точки по сфере Фибоначчи, дыхание+вращение, отключение при reduced-motion); вертикаль собеседников у правого края (мессенджер-паттерн: живые орбы + бейджи находок + пунктирный «+ слот»); оверлей разговора (карта в blur+dim позади; большой орб слева, колонка тем разговоров с привязкой к целям, переписка, ввод «говори или пиши» с микрофоном); «шёпот» (последняя реплика одной строкой у орба в покое); баллоны находок у иконок агентов.
- **Выходы (зафиксировать как принципы навигации, не реализация):** клик по орбу в разговоре → карточка агента «как персонаж игры» (навыки, знания, модель-«мозг», аватар — связка с Management_Model §5–§7, Университетом и шкалой зрелости/автономности); зум/клик в узел цели → процесс достижения и уровень декомпозиции ниже; глубокий зум → рабочее пространство сущности (Workspace_Platform) с дашбордами/документами/отчётами как возможностями (Dashboard_Builder, Report_Builder). Дашборды и отчётность — НЕ пункты меню, а содержимое пространств.
- **Язык роя точек:** карта целей остаётся 2D (рабочая поверхность, читаемость приоритетна — решение владельца); рой точек = язык живого и неструктурированного: орбы (сейчас), «туман-рой» — неоформленные идеи как россыпь тлеющих точек, конденсирующихся в узел при оформлении (ближнее направление), режим «галактика» — смысловая кластеризация больших массивов сущностей: артикулы, знания, находки (дальнее направление, не строить до появления массивов). 2.5D-параллакс слоёв — опция вкуса, не обязательство.
- Явная строка: Часть II (as-built Terminal Strategist) остаётся истиной ТЕКУЩЕГО кода до рестайл-прохода №20; после рестайла Часть II переписывается под новый as-built, Часть III становится её источником. В Части I пометить пункты, которые v2 заменяет (моно-«терминальность» D1-тона), не переписывая историю.

### 3. Новый док `docs/full-vision/02_Product/User_Scenarios.md`
Шесть ролевых сценариев (дистиллировать из слов владельца, сохранив суть; сарказм окультурить, но не потерять):
1. **Основатель** — вектор и стратегия; системные проблемы и успехи; докопаться «до винтика» — все этапы всех целей. → карта целей + уровни масштаба (§3 Части I) + сигналы сквозных агентов.
2. **Топ-менеджер** — системность, управляемость, минимум хаоса. → увязка целей, циклы, перегрузы (ADR-0004), Decision Center.
3. **Руководитель отдела** — «получу ли премию за квартал/год и кто в этом виноват»: план/факт своего узла и вклад подчинённых. → ADR-0005 (факт KPI из ресурсных блоков), декомпозиция узла.
4. **Исполнитель (дизайнер)** — «плевать на высшие материи»: сколько роликов запилить, агент помогает пилить, быстрая организация в архив и автозагрузка на каналы. → пространство без стратегического слоя: задачи + агент-помощник + автоматизация рутины.
5. **Менеджер маркетплейса** — «я пилот, агенты — штурманы»: метрики и воронки магазина, где просадка; обученные агенты (аналитик-сводки, дизайнер-инфографика под праздники, перформанс-менеджер рекламы); справочники артикулов (характеристики не теряются при миграции между маркетплейсами), поставки, юнит-экономика каждого SKU. → лучший кандидат на первое «пространство из блоков» (Продажи + Товары), аргумент за вехи ресурсных блоков.
6. **Бухгалтер (анти-сценарий)** — «не трогайте меня»: сдать отчётность вовремя, до копейки; ИИ ошибается, проверять = делать самому. → зафиксировать честно как ограничение Unified Workforce: есть роли, где человек — контролёр агента, а не наоборот; стандартизированная отчётность = Report_Builder, направление.

Каждому сценарию — строка «какие механики уже есть / какие вехи это требует». Это ориентиры для приоритизации, НЕ задачи.
