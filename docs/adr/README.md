# Architecture Decision Records (ADR)

Журнал архитектурных решений проекта Decision Center. Здесь фиксируются значимые и трудно-обратимые решения: выбор технологий, структурные принципы бэкенда/фронтенда, отступления от спеков.

## Правила ведения

1. Один файл — одно решение: `NNNN-краткий-слаг.md` (номер сквозной, по порядку).
2. Шаблон — [0000-template.md](0000-template.md): Контекст / Решение / Статус / Последствия.
3. ADR не редактируются задним числом: устаревшее решение получает статус «Заменено (ADR-XXXX)» и ссылку на новое.
4. Язык — русский (правило репозитория, см. `docs/full-vision/AGENTS.md`); устоявшиеся технические термины — как принято.
5. ADR фиксируют **реализационные** решения; продуктовые требования живут в `docs/full-vision/02_Product/PRD.md` и `Management_Model.md` — ADR им не противоречат, а ссылаются на них.
