# Технические требования

## 1. Общие требования

### 1.1. Цель документа

Данный документ описывает технические требования к системе AI Company Simulator на основе разделов PRD: 30 (AI-рантайм), 41 (Самооптимизация), 43 (Здоровье), 44 (Симуляция), 45 (Бизнес-влияние), 46 (Операционный интеллект), 47 (Обратная связь).

### 1.2. Основные компоненты системы

- AI-рантайм для выполнения моделей и агентов
- Система самооптимизации
- Мониторинг здоровья системы
- Среда симуляции
- Оценка бизнес-влияния
- Операционный интеллект
- Система обратной связи

## 2. Требования к инфраструктуре

### 2.1. Локальные LLM

Для обеспечения работы AI-агентов и моделей используются следующие инструменты:

#### 2.1.1. Ollama
- **Назначение:** запуск моделей до 20B параметров
- **Преимущества:** простота установки, низкие требования к ресурсам, поддержка множества моделей
- **Рекомендуемые модели:** Llama 3 8B, Mistral 7B, Gemma 7B

#### 2.1.2. vLLM
- **Назначение:** запуск моделей от 70B параметров и выше
- **Преимущества:** высокая производительность, поддержка пакетной обработки, оптимизация памяти
- **Рекомендуемые модели:** Llama 3 70B, Mistral Large

### 2.2. Требования к серверному оборудованию

#### 2.2.1. GPU
- **Минимальные требования:** 16 ГБ VRAM (NVIDIA RTX 4080 или аналоги)
- **Рекомендуемые требования:** 48 ГБ VRAM (NVIDIA A6000, RTX 6000 Ada или аналоги)
- **Поддержка CUDA 11.8+ и cuDNN 8.9+**

#### 2.2.2. Оперативная память (RAM)
- **Минимальные требования:** 32 ГБ
- **Рекомендуемые требования:** 64-128 ГБ для работы с большими моделями

#### 2.2.3. Хранилище
- **Тип:** NVMe SSD
- **Минимальный объём:** 512 ГБ
- **Рекомендуемый объём:** 1-2 ТБ для хранения моделей и данных

### 2.3. Сетевая инфраструктура

- **Пропускная способность:** минимум 1 Гбит/с
- **Задержка:** не более 10 мс между компонентами системы
- **Резервирование:** дублирование каналов связи для критических компонентов

## 3. Интеграции с внешними системами

### 3.1. Гарант Коннект (API)

- **Назначение:** интеграция с системой документооборота и юридически значимыми документами
- **Протокол:** REST API
- **Формат данных:** JSON
- **Аутентификация:** API-ключ + OAuth 2.0
- **Основные операции:**
  - Загрузка документов
  - Проверка статуса
  - Получение подписанных документов

### 3.2. MPSTATS (API)

- **Назначение:** получение статистических данных и метрик для анализа
- **Протокол:** REST API / GraphQL
- **Формат данных:** JSON
- **Аутентификация:** API-ключ
- **Основные операции:**
  - Получение агрегированных данных
  - Запрос исторических метрик
  - Экспорт отчётов

### 3.3. Google Docs / Sheets (API)

- **Назначение:** работа с документами и таблицами Google
- **Протокол:** REST API
- **Формат данных:** JSON
- **Аутентификация:** OAuth 2.0
- **Основные операции:**
  - Создание и редактирование документов
  - Импорт/экспорт данных
  - Совместная работа над документами

## 4. Работа с документами

### 4.1. Excel (openpyxl, pandas)

- **Назначение:** генерация отчётов по шаблонам, экспорт данных, обработка таблиц
- **Библиотеки:**
  - **openpyxl:** чтение и запись Excel-файлов (.xlsx)
  - **pandas:** обработка и анализ табличных данных
- **Функциональность:**
  - Создание отчётов по шаблонам
  - Экспорт данных из системы
  - Импорт данных из Excel
  - Форматирование и стилизация

### 4.2. Форматы документов

- **Входные:** .xlsx, .xls, .csv
- **Выходные:** .xlsx, .pdf, .html
- **Шаблоны:** .xltx с предустановленными стилями и формулами

## 5. Безопасность

### 5.1. RBAC (Role-Based Access Control)

#### 5.1.1. Роли

| Роль | Описание | Права |
|------|----------|-------|
| **Владелец** | Полный доступ к системе | Все операции, управление пользователями, настройка системы |
| **Руководитель** | Управление командой и проектами | Просмотр всех данных, управление проектами, назначение задач |
| **Сотрудник** | Выполнение задач | Доступ к своим данным, выполнение задач, просмотр отчётов |
| **Настройщик** | Конфигурация системы | Настройка параметров системы, управление интеграциями, обновление моделей |

#### 5.1.2. Матрица доступа

| Операция | Владелец | Руководитель | Сотрудник | Настройщик |
|----------|----------|--------------|-----------|------------|
| Просмотр данных | ✅ | ✅ | ✅ (свои) | ✅ |
| Редактирование данных | ✅ | ✅ | ✅ (свои) | ❌ |
| Управление пользователями | ✅ | ❌ | ❌ | ❌ |
| Настройка системы | ✅ | ❌ | ❌ | ✅ |
| Управление интеграциями | ✅ | ❌ | ❌ | ✅ |
| Аудит | ✅ | ✅ | ❌ | ❌ |

### 5.2. Шифрование данных

#### 5.2.1. Шифрование в состоянии покоя
- **Базы данных:** AES-256
- **Файлы:** AES-256
- **Резервные копии:** AES-256

#### 5.2.2. Шифрование в транзите
- **TLS 1.3** для всех сетевых соединений
- **HTTPS** для веб-интерфейса
- **SSH** для удалённого доступа

#### 5.2.3. Управление ключами
- **HSM** (Hardware Security Module) для хранения ключей
- **Ротация ключей:** каждые 90 дней
- **Резервное копирование ключей** в защищённом хранилище

### 5.3. Аудит действий

#### 5.3.1. Логирование
- Все действия пользователей записываются в лог
- Логи содержат: пользователь, действие, время, IP-адрес, результат
- Логи хранятся в Elasticsearch с retention 1 год

#### 5.3.2. Мониторинг
- Отслеживание подозрительной активности
- Уведомления при нарушении политик безопасности
- Регулярные проверки соответствия

## 7. Требования к производительности

### 7.1. Время отклика

| Компонент | Максимальное время | Целевое время |
|-----------|-------------------|---------------|
| API запросы | 500 мс | < 200 мс |
| Инференс LLM | 5 с | < 2 с |
| Загрузка отчётов | 10 с | < 3 с |
| Поиск | 2 с | < 500 мс |

### 7.2. Пропускная способность

| Компонент | Минимум | Целевой показатель |
|-----------|---------|-------------------|
| API (RPS) | 100 | 500+ |
| Инференс (токен/с) | 50 | 200+ |
| Обработка событий | 1000/мин | 10000/мин |

### 7.3. Доступность

- **SLA:** 99.9% (менее 8.76 часов простоя в год)
- **RPO:** не более 15 минут
- **RTO:** не более 1 часа

## 8. Требования к масштабированию

### 8.1. Горизонтальное масштабирование

- **Веб-серверы:** до 10 инстансов
- **Бэкенд-сервисы:** до 20 инстансов
- **База данных:** репликация master-slave
- **Очереди:** кластер RabbitMQ

### 8.2. Вертикальное масштабирование

- **Добавление GPU:** до 8 GPU на сервер
- **Увеличение RAM:** до 2 ТБ на сервер
- **Увеличение хранилища:** до 10 ТБ
