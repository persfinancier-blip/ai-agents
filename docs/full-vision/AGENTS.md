# Инструкции для AI-агентов

Этот файл определяет правила работы всех AI-агентов (Claude Code, Cursor, Codex и др.) с документацией проекта AI Company Simulator. Вся документация видения лежит в `docs/full-vision/` — пути ниже даны от корня репозитория.

## Обязательные шаги перед началом любой задачи

1. Прочитай `README.md` (корень репозитория), чтобы понять структуру проекта и актуальные милстоуны.
2. Прочитай `docs/full-vision/INDEX.md`, чтобы найти нужный документ — это единственная точка навигации.
3. Если задача связана с изменением требований — сначала обратись к `docs/full-vision/02_Product/PRD.md` (это источник истины).
4. Если задача касается модели управления (карты целей, роли, увязка, навыки/компетенции, офисные агенты) — обратись к `docs/full-vision/02_Product/Management_Model.md`.
5. Если ты должен создать новый документ или внести изменения, убедись, что они согласуются с Философией продукта (раздел 10 PRD).

## Приоритет документов

1. `02_Product/PRD.md` — главный. Любой конфликт между документами разрешается в пользу PRD.
2. `02_Product/Management_Model.md` — источник истины по модели управления фирмой; приоритет сразу после PRD.
3. `02_Product/MVP_Scope.md` — определяет, что входит в первый релиз.
4. `05_Architecture/Technical_Requirements.md` — технические ограничения.
5. `06_Roadmap/Roadmap.md` — планы развития.

Активные спеки текущего вертикального среза (Decision Center): `04_Simulation/Decision_Center.md`, `05_Architecture/Entity_Platform.md`, `09_Design_System/Visual_Reference.md`.

## Правила изменений

1. Не изменяй PRD без явного запроса от человека.
2. При добавлении нового документа обнови `docs/full-vision/INDEX.md`.
3. Все новые документы должны быть на русском языке (если не указано иное).
4. Используй Markdown с заголовками (##, ###) и, где необходимо, диаграммы Mermaid.

## Ответственность

Ты — AI-агент, помогающий в разработке. Твоя задача — следовать этим инструкциям и запрашивать уточнения у человека, если что-то неясно.
