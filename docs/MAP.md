## Каноны

### PRD — `docs/full-vision/02_Product/PRD.md`
- 1. Видение продукта — три принципа платформы.
- 2. Описание проблемы — фрагментация, вовлечение, разрыв стратегия/исполнение.
- 3. Ценностное предложение — что получает каждый тип пользователя.
- 4. Цели продукта — 10 продуктовых целей.
- 5. Метрики успеха — целевые числа вовлечённости/бизнеса.
- 6–9. Аудитория, персонажи, JTBD, путь пользователя.
- 10–11. Философия и принципы продукта.
- 12–13. Основной цикл продукта; концепция симуляции компании.
- 14–19. Модель сущностей: типы, атрибуты, жизненный цикл, граф, двойник, наследование, сотрудничество, кросс-сущностный интеллект.
- 20–27. Живое предприятие: философия, эволюция, зрелость, память, адаптивный интеллект.
- 28–31. Единая рабочая сила и цифровой сотрудник: концепция, типы, AI-рантайм, жизненный цикл.
- 32–38. Обучение, KPI, оценка эффективности, память/знания, навыки vs компетенции, AI-управление.
- 39–40. Маркетплейс, геймификация.
- 41–47. Самооптимизирующееся предприятие: измеримость, движок оптимизации, здоровье организации, прогнозная симуляция, бизнес-влияние, операционный интеллект, обратная связь.
- 48–54. Функциональный объём (MVP/желательно/можно/будущее), вне области, риски, допущения, открытые вопросы, критерии успеха, заключение.

Полное оглавление с одной строкой на раздел — в TOC самого файла (сразу после заголовка).

### Management_Model — `docs/full-vision/02_Product/Management_Model.md`
- 0. Назначение — что канон, что открытый вопрос.
- 1. Два типа карт (Ф1) — дерево-декомпозиция vs воркфлоу-навык.
- 2. Роли и поток управления (Ф2) — собственник → менеджмент → исполнитель → юнит.
- 3. Увязка — мягко-жёсткая (Ф3) — «система предлагает, человек утверждает», ресурсный слой.
- 4. Увязка через опрос офисных агентов (Ф4) — поздний срез M7–M8.
- 5. Навык ≠ Компетенция (Ф5) — разведение терминов.
- 6. Всё есть Сущность (Ф6) — маппинг на Entity Platform.
- 7. Увязка: статус открытых вопросов (Ф7).
- 8. Расхождения с PRD (приоритет у PRD).
- 9. KPI-сущность и связи целей (Ф8) — KPI, индивидуальные/командные цели, циклы, составной KPI.

Полное оглавление с одной строкой на раздел — в TOC самого файла.

## Код

### `backend/app/` (FastAPI + SQLAlchemy async + Alembic)
- `api/v1/` — HTTP-роуты.
- `models/` — ORM-сущности: `entity`, `goal`, `unit`, `unit_group`, `kpi`, `kpi_factor`, `kpi_link`, `kpi_link_cycle`, `decision`, `board_opinion`, `scenario`, `status_log`.
- `services/` — бизнес-логика по одному сервису на сущность (`goal_service`, `kpi_service`, `unit_service`, `board_service`, `decision_service` и др.).
- `schemas/` — Pydantic-схемы запросов/ответов.
- `llm/` — единственное место импорта LLM SDK (провайдер-изоляция); `LLM_PROVIDER=stub` локально.
- `db/`, `config.py`, `main.py` — подключение БД, настройки, точка входа.

### `frontend/src/` (React 19 + TS + Vite)
- `os/` — прототип Command Center: `CommandPanel.tsx` (панель), `GoalCanvas.tsx`/`GoalCard.tsx`/`GoalPopup.tsx` (карта целей), `UnitsPanel.tsx` (юниты), `ProcessMap.tsx` (BPMN-подобный граф), `AdvisorOrb.tsx` (советник), `data.ts`/`goals.ts`/`units.ts`/`goalTree.ts`/`goalFormat.ts` (демо-данные и форматирование), `os.css` (дизайн-токены — источник Visual_Reference Части II).
- `App.tsx`, `api.ts`, `types.ts`, `main.tsx` — точка входа, HTTP-клиент, общие типы.

## Процесс

- `docs/PROCESS.md` — протокол полного цикла разработки (роли, жизненный цикл задачи, DoD-гейт, откат).
- `.claude/rules/*` — зональные правила (backend/frontend/docs/commits/github-automation).
- `prompts/` — конвейер промптов: `prompt-NN-*.md` (продукт), `prompt-ops-NN-*.md` (инфраструктура); выполненные — в `prompts/_done/`.
- `docs/DEVLOG.md` — индекс журнала прохода (таблица частей + одна строка на запись); сами записи — в `docs/devlog/part-NN.md`.
- `BACKLOG.md` — задачи на будущее.
- `docs/adr/` — архитектурные решения (ADR), индекс — `docs/adr/README.md`.
